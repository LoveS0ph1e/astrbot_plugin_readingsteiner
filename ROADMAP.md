# ReadingSteiner · 路线图 (Roadmap)

> 本文档记录 ReadingSteiner 的演进方向与待办，供贡献者与使用者了解项目走向。
> 不代表承诺的时间表；随项目演进更新。
>
> - 已发布变更见 [CHANGELOG.md](CHANGELOG.md)；当前能力、配置与已知边界见 [README.md](README.md)。
> - 版本系列代号 **Crepuscule**（源自莫扎特《魔笛》）。

本项目依赖三层，路线图据此分区：

| 层 | 角色 |
|---|---|
| **ReadingSteiner 插件**（本仓库） | AstrBot 钩子、记忆注入/归档、命令、身份隔离与隐私分层 |
| **EverAlgo**（上游记忆算法库） | 用户画像 / 情景的抽取算法与 prompt——插件的记忆质量很大程度取决于此 |
| **EverOS**（上游记忆引擎 / 服务） | 持久化存储、检索 API、离线编排 |

插件运行中沉淀出的若干上游改进，将以 PR 形式回馈 EverAlgo / EverOS（见第二、三节）。

---

## 一、插件 · 近期发布

以下能力已实现，待统一版本发布（详见 CHANGELOG `[Unreleased]`）：

- **群聊按发送者隔离记忆**：群聊中每个成员使用独立的引擎会话键，避免多人对话被合并抽取而
  导致跨用户记忆串线、以及助手人设倒灌进用户画像。
- **相关情景记忆完整句注入**：注入的情景文本按句末标点收束（中英双语），不再在句中截断。
- **记忆遗忘（抑制）**：管理员 `/epk forget`，在读路径抑制指定记忆，可逆。

发布相关收尾：

- **版本与发布**：统一版本号（`metadata.yaml` / 插件注册元数据 / `CHANGELOG`），整理发布说明并打 tag。
- **可观测性**：归档 / flush 成功时输出 INFO 日志，便于肉眼确认记忆已落库（当前成功路径静默）。

## 二、上游贡献 · EverAlgo PR

### PR-A：ProfileExtractor 抽取质量（归属 + 抽象 + 时间粒度）

将抽取从「目标盲」改为「定向」，并提升描述的持久性：

- **定向归属**：把目标用户 id 传入抽取 prompt，使多说话人（群聊）对话中的事实归属到正确的人；
  明确排除「助手自身的身份 / 人设」与「其他参与者的发言」进入目标用户画像。
  （上游已有面向群聊的 prompt 草案，但未接入现役抽取器；本 PR 即完成接入。）
- **持久抽象**：画像 `description` 写成「持久概括」而非「逐事件流水账」，避免把一次性事件、
  细粒度时间堆进画像。
- **时间粒度粗化**：抽取输入的对话时间戳粗化到日期级，消除秒级噪声并回避时区误读。
- 涉及：`user_memory/profile.py`、`prompts/{en,zh}/profile.py`（含 COMPACT 一致性）。

### PR-B：情景 summary 完整句

- **现状**：当抽取未产出独立 `summary` 时，回退为 `content[:N]` 的字符级硬截，常在句中斩断。
- **改动**：回退改为按句末标点收束（或令抽取直接产出独立 `summary` 字段）。
- 涉及：`user_memory/episode.py`（及相应 prompt）。
- 备注：插件侧已在注入层对截断 summary 做了兜底（完整句收束），本 PR 进一步在存储源头修正。

### （可选）冗长自愈

为 ProfileExtractor 增加「冗长触发的再抽象（REFINE）」策略，并在代码层加忠实度闸
（条目守恒、关键实体不丢则采纳，否则弃用）。边际价值较低，视需要再开。

## 三、上游方向 · EverOS

- **真删除端点**：为引擎增加画像 / 情景的删除 API，使插件能真正擦除而非仅在读路径抑制——
  这是 `/epk forget` 的「另一半」。需上游设计，体量较大。
- **prompt 用户层覆盖接入抽取**：将引擎的 prompt 槽位（用户层覆盖）接入画像抽取，使 prompt
  可经配置定制，减少对算法库的直接修改。待评估。
- **（记录）情景索引口径**：观察到部分用户的情景 `get` / `search` 返回与磁盘记录不一致，留待核查。

## 四、插件 · 后续增强

- **细粒度可见性分层**：当前群聊隐私为粗粒度（群聊丢弃情景细节）；规划按「用户 × 可见性类目」
  的细粒度规则表（`visibility` 模块已预留扩展点）。
- **管理 WebUI（姊妹项目 Eyes of Priestess，版本系列 Sarastro）**：画像、永恒铭契、遗忘规则的
  可视化查看与手动编辑。
- **画像质量校验增强**：当前为规则底线（结构缺陷、缺证据＝疑似幻觉）；「证据存在但推断错误」的
  语用误判需 LLM judge，尚未实现。

## 五、已知边界 · 记录备查

- **群聊实时上下文的多说话人归属**：引擎按发送者隔离的*长期记忆*已正确归属；但模型每轮接收的
  *近期群对话上下文*是多说话人的，生成时仍可能把不同人的当下话题归并到当前对话者。此属对话
  上下文 / 人格 prompt 层，不在记忆插件职责内，记录备查。
- **删除＝遗忘（抑制）而非擦除**：见 README「已知边界」；真擦除待第三节「真删除端点」。
- **最终一致窗口**：flush 后检索、以及画像改写后的生效，均可能有短暂延迟（见 README）。
- **画像质量校验的盲区**：见第四节「画像质量校验增强」。
